# 제어 노드 설계 (control_node)

작성: 2026-06-03. 코드 작성 전 제어 로직 확정용 설계 문서.
대상 로봇: 메카넘 4륜 + 리프트(승강) + 상단 yaw 스테이지(꼭대기에 OAK-D + 촬영카메라).

---

## 0. 역할 경계 (중요)

- **제어 노드(이 문서)**: 추적 정보를 받아 **자유도 속도 명령**만 낸다.
  몸체 vx/vy/yaw, 리프트 vz, 상단 yaw rate. **모터 개수·추력은 모른다.**
- **드라이버 노드(팀원)**: 속도 명령 → 모터 6개(메카넘4 + 리프트1 + 상단1) 추력 매핑.
- **supervisor(상태기계, 정우)**: 손동작 이벤트 → 모드 결정 → `/control_mode` 발행.
- **손동작 노드(MediaPipe, 팀원과 분담)**: OAK-D RGB로 제스처 인식 → `/gesture` 발행.

제어 노드는 `/control_mode`만 본다. "왜 그 모드인지"(제스처)는 모른다.

---

## 1. 좌표계 약속

ROS REP-103 + OAK 출력 기준. 부호를 여기서 못박는다.

| 대상 | 축 / 부호 |
|------|-----------|
| 몸체 | x 전방(+), y 좌측(+), yaw CCW(+) |
| 카메라(OAK) | spatial_x 우측(+), spatial_y 아래(+), spatial_z 전방(+) |
| azimuth | `atan2(spatial_x, spatial_z)` → 주인이 **우측이면 양수** |
| 상단 yaw rate | CCW(+) = 좌회전 |

주의: 카메라 우측(azimuth>0)으로 카메라를 돌리려면 시계방향 = CCW 음수.
→ 상단 yaw 제어식 부호가 음수가 된다.

---

## 2. 출력 메시지: ControlCmd.msg

전용 메시지로 6자유도를 의미대로 싣는다(안2 확정).

```
# ControlCmd.msg
std_msgs/Header header
float32 body_vx        # m/s, 몸체 +전방
float32 body_vy        # m/s, 몸체 +좌측
float32 body_yaw_rate  # rad/s, +CCW(좌회전)
float32 lift_vz        # m/s, +위
float32 top_yaw_rate   # rad/s, +CCW(좌회전)
```

→ 드라이버 노드(팀원)가 이 메시지를 구독해 모터 추력으로 매핑.

---

## 3. 입력

| 토픽 | 타입 | 내용 |
|------|------|------|
| `/owner_pose` | OwnerPose | is_detected, spatial_x/y/z, azimuth, distance, confidence, track_id |
| `/control_mode` | std_msgs/Int32 | 주행 모드 (supervisor가 결정) |
| `/lift_height` | std_msgs/Float32 | 리프트 현재 높이 [m] (있으면 P제어, 없으면 적분 폴백) |
| `/follow_target` (선택) | geometry_msgs/Vector3 | 목표 오프셋(거리 D, 방위, 높이) |
| `/height_cmd` (미래) | std_msgs/Float32 | 손동작 노드가 주는 리프트 목표 높이 |

상단 yaw 엔코더(θ_head)는 **받지 않는다**. 아래 4.2 참조 — 불필요.

## 4. 제어 법칙

### 4.1 상단 yaw — 빠른 visual servoing (항상 동작)

목표: `azimuth → 0` (OAK-D를 주인에 락온). 트래킹 노드 정보만으로 완결.

```
e_az = azimuth                                  # 목표 = 0
top_yaw_rate = -(Kp_yaw*e_az + Ki_yaw*∫e_az + Kd_yaw*ė_az)
top_yaw_rate = clamp(top_yaw_rate, -w_top_max, +w_top_max)
# deadzone: |e_az| < az_dead 이면 0, 적분 리셋
# soft limit: 누적각 추정이 ±theta_soft_max 근처면 바깥방향 포화 (언와인딩 대비, 5장)
```

owner가 보이는 한 항상 돈다. 몸이 어떻게 움직이든 카메라는 주인에 고정.

### 4.2 몸체 yaw — 몸을 OAK 방향에 천천히 정렬 (모드1)

핵심 통찰: **"몸체를 OAK 방향에 맞춘다" = "상단 yaw가 더 일할 필요 없게 만든다".**
그 신호는 **상단 yaw rate 출력 그 자체**다. θ_head 적분·엔코더 불필요.

머리가 어느 쪽으로 일하면, 몸도 그쪽으로 천천히 따라 돈다:

```
body_yaw_rate = Kp_align * top_yaw_rate         # 머리 따라 몸 정렬 (느리게)
body_yaw_rate = clamp(body_yaw_rate, -w_body_max, +w_body_max)
# Kp_align < 1, w_body_max < w_top_max  → 머리는 빠르고 몸은 느리게 (루프 분리)
```

정상상태: 몸이 정면을 맞추면 azimuth→0 유지에 필요한 top_yaw_rate도 0 →
body_yaw_rate도 0. 동시에 누적각도 0 부근으로 풀려 2바퀴 한계에도 유리.

> 대안(미채택): top_yaw_rate 대신 azimuth를 직접 정렬 오차로 써도 된다.
> 단 그러면 머리와 몸이 같은 신호로 경쟁할 수 있어, "머리 출력을 몸이 따라간다"는
> 위 방식이 루프 분리상 더 안전. 실주행에서 떨림 나면 신호원 재검토.

### 4.3 몸체 vx/vy — 거리 유지 (모드1)

몸이 정면 정렬되면 주인은 몸체 정면 방향 → 거리만 맞추면 됨(vy는 0).

```
e_dist = distance - D_target                    # +면 주인이 멀다 → 전진
body_vx = clamp(Kp_dist * e_dist, -v_rev_max, +v_fwd_max)
body_vy = 0                                      # 모드1: 정렬로 처리, 좌우이동 불필요
# deadzone: |e_dist| < d_dead 이면 0, 적분 리셋
# slew-rate 제한: 메카넘 미끄럼/부하 보호
```

> 정렬이 덜 된 과도구간에선 주인이 정면에서 약간 벗어나 있어 vx가 비스듬히 작용하지만,
> 상단 yaw가 락온을 유지하고 몸이 곧 정렬되므로 수렴. 떨림 시 D_target deadzone로 흡수.

### 4.4 리프트 vz — 정지 시 높이 제어 (게이팅)

하드웨어 안정성: 주행 중 승강 금지. 정지(또는 compose 모드) 시에만.

```
moving = norm(body_vx, body_vy) > v_stop_thresh
if moving and mode != COMPOSE:
    lift_vz = 0
else:
    lift_vz = clamp(Kp_lift * (h_target - h_now), -vz_max, +vz_max)
    # (선택) 세로 오토프레이밍: spatial_y로 주인 수직중앙 유지
    #   카메라 올리면 주인이 프레임에서 내려감(sy 증가) → sy 줄이려면 내려야 함
    #   lift_vz = -Kp_vframe * (spatial_y - sy_target)
    lift_vz = enforce_stroke_limits(lift_vz, z_min, z_max, h_now)   # 행정 끝 보호
```

h_target은 손동작 노드(미래)가 `/height_cmd`로 채운다.

## 5. 상단 yaw 언와인딩 (추가 기능 — 지금은 자리만)

상단 yaw 가동범위 최대 ~2바퀴(연속회전 불가). 시연 중 꼬임 방지 필요.
**로직 구현은 나중 세션.** 지금은 인터페이스 자리만 마련:

- 파라미터 `theta_soft_max`(예: ±1.8바퀴) 선언만. 지금은 큰 값(사실상 무한).
- 누적각 추정 `theta_accum += top_yaw_rate * dt` — **제어엔 안 쓰고 모니터링/트리거용**.
  드리프트 나도 무관(언와인딩은 정밀도 불필요한 휴리스틱).
- 후보 로직(나중): 정지+주인 정면일 때 한 바퀴 거꾸로 / 주기적 역회전 / 시연 동선 휴리스틱.

## 7. 안전 / 엣지 케이스

- **owner 미탐지/타임아웃**(is_detected=false 또는 watchdog): 몸체·리프트 즉시 0.
  상단 yaw는 선택적으로 마지막 azimuth 부호로 천천히 search 후 정지(soft limit 내).
- **저신뢰도**(confidence 낮음): 게인 축소 또는 hold.
- **deadzone** 3종(azimuth / 거리 / 높이) — 떨림 방지.
- **출력 클램프**: v_fwd_max, v_rev_max, w_top_max, w_body_max, vz_max.
- **slew-rate 제한**: 메카넘·리프트 부드럽게.
- **리프트-주행 상호배제**: 4.4 게이팅.
- **dt 보호**: dt<=0 또는 dt>1.0 이면 기본값(예 0.02s).
- **고정주기 출력**: 검출(~19Hz)에 묶지 말고 50Hz 타이머에서 최신 owner_pose 캐시해 제어.
  → 드라이버 노드 입장에서 더 매끄러움.

## 8. 손동작 명령 레이어 (supervisor, 제어 노드 밖)

제어 노드에 넣지 않는다. 상위 상태기계가 모드를 지배한다.

```
/gesture (손동작 노드 → supervisor)
   "thumbs_up"    # 명령 진입
   "fingers_1"    # 선택지 1
   "fingers_2"    # 선택지 2
   "thumbs_down"  # 확정/종료
   ...
```

상태 흐름:
1. RUNNING: 제어 노드 구동 중.
2. 따봉 수신 → HOLD: supervisor가 `/control_mode=0(IDLE)` 발행 → 제어 정지.
3. 디스플레이에 선택지 표시 → 손가락 개수로 모드 선택(SELECTING).
4. 아랫따봉 확정 → 선택 모드를 `/control_mode`로 주입 → RUNNING 재개.

제어 노드는 `/control_mode` 변화만 보고 멈추고/재개. 제스처는 모름.

## 10. 제어 노드 골격 (수도코드)

```
class ControlNode:
  init():
    declare_params(...)                          # 9장
    sub /owner_pose    -> cache owner, stamp last_owner_time
    sub /control_mode  -> set mode, reset_integrators, publish_zero
    sub /lift_height   -> cache h_now
    sub /follow_target(opt) -> set d_target 등
    sub /height_cmd(미래)   -> set h_target
    pub /control_cmd   (ControlCmd)
    timer(ctrl_rate=50Hz) -> control_step()

  control_step():
    out = ControlCmd()                           # 전부 0 초기화
    if mode == IDLE: publish(out); return
    if (now - last_owner_time) > owner_timeout or not owner.is_detected:
        reset_integrators()
        out.top_yaw_rate = search_or_zero(last_az_sign)   # 선택, soft limit 내
        publish(out); return                     # body/lift = 0

    dt = guard_dt(now - last_step_time)

    # --- 4.1 상단 yaw (항상) ---
    e_az = owner.azimuth
    out.top_yaw_rate = pid_yaw(e_az, dt)         # deadzone, clamp, soft limit
    last_az_sign = sign(e_az)
    theta_accum += out.top_yaw_rate * dt         # 모니터링용(제어 미사용)

    # --- 4.2 몸체 yaw + 4.3 거리 (모드별) ---
    if mode == FOLLOW:
        out.body_yaw_rate = clamp(kp_align * out.top_yaw_rate, -w_body_max, w_body_max)
        e_dist = owner.distance - d_target
        out.body_vx = clamp(kp_dist * e_dist, -v_rev_max, v_fwd_max)  # deadzone, slew
        out.body_vy = 0
    elif mode == ORBIT:
        orbit_law(...)                           # TODO 스텁
    elif mode in (STATIONARY, COMPOSE):
        pass                                     # body 전부 0

    # --- 4.4 리프트 (게이팅) ---
    moving = norm(out.body_vx, out.body_vy) > v_stop_thresh
    if moving and mode != COMPOSE:
        out.lift_vz = 0
    else:
        out.lift_vz = lift_law(h_target, h_now, owner.spatial_y)      # limits

    publish(out)
    last_step_time = now
```
