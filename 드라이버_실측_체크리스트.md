# 드라이버 실측 캘리브 체크리스트 (STM32+모터 연결 후, 실기에서)

> 코드(bridge↔펌웨어↔control)는 계약상 검증됨. 아래는 "하드웨어 실측이 안 되면
> 틀어지는" 항목들. 순서대로 한 번씩 잡으면 odom/주행이 맞는다.
> 기동: solcam.sh + driver_bridge. teleop 으로 순수 자유도만 줘서 확인.

## 0. 사전
- STM32 USART1 ↔ Jetson USB-시리얼 연결, `ls /dev/ttyUSB* /dev/ttyACM*` 확인.
- driver_bridge 실행:
    ros2 run ros2_driver_bridge driver_bridge --ros-args \
      -p port:=/dev/ttyUSB0 -p baud:=115200
  (권한 막히면: sudo usermod -aG dialout $USER 후 재로그인)
- 토픽 확인: `ros2 topic hz /odom` (≈100Hz), `ros2 topic echo /top_yaw_state`.
- ★바퀴 들고(혹은 받침대) 시작 — 부호 틀리면 갑자기 달려나갈 수 있음.

## 1. ★encoder_cpr (1순위 — 안 맞으면 odom 거리/속도 전부 스케일 오류)
- 바퀴 하나를 손으로 정확히 1회전 시키고 `/odom` 또는 enc 델타 누적을 본다.
- 또는: 알려진 거리(예 1.0m) 직선 주행시키고 odom x 가 1.0 나오게 cpr 조정.
- cpr = (휠1회전 카운트). 쿼드라처(x4) + 기어비(JGB37-520) 포함 실측값.
- 적용: driver_bridge -p encoder_cpr:=<측정값>  (기본 1320 은 placeholder!)

## 2. 휠 순서/부호 (encoder_order / encoder_signs)
- 바퀴 1개씩 손으로 +방향(전진 굴림) 돌리며 어떤 enc 가 +로 증가하는지 확인.
- 매핑: enc1~4(tim1/2/4/8) → 물리 [FL,FR,RL,RR]. 기본 order [0,1,2,3].
- 전진 굴림에 −로 가는 휠은 encoder_signs 에서 해당 칸 −1.
- 적용: -p encoder_order:="[...]" -p encoder_signs:="[...]"

## 3. ★DOF 방향 (teleop 으로 — 펌웨어 메카넘 부호 확인)
모드0(IDLE)에서 teleop 으로 순수 1축씩:
- +vx(전진) 명령 → 로봇이 앞으로? (뒤로 가면 펌웨어 휠 배선/부호 반대)
- +vy(좌) 명령 → 왼쪽으로 게걸음?
- +wz(좌회전) 명령 → 반시계로 제자리 회전?
하나라도 반대면 펌웨어 Mecanum_SetVelocity 의 해당 항 부호 또는 모터 배선 교정.
(control 의 좌표 규약: +x전방 +y좌 +CCW. 이게 실제와 같아야 FOLLOW 가 맞게 감.)

## 4. 상단 yaw 0점 / 부호
- 부팅 시 yaw_current_steps=0 = "그 순간 카메라 방향"이 0. 시작 전 카메라를
  몸체 정면에 맞춰두고 켜기(또는 영점 맞추는 절차 정하기).
- +yaw_target 명령 → 카메라가 좌(CCW)로? control 규약은 +CCW. 반대면
  펌웨어 Yaw_SetTarget 의 DIR SET/RESET 두 줄 swap (YAW_점검.md 4번).
- 트래킹과 연동 확인: 주인을 우측에 두면 카메라가 우측으로 따라가야(주인 락온).
  반대로 가면(주인서 멀어짐) yaw 부호 반대 → 위 swap.

## 5. 리프트 방향/행정
- +lift_height_target → 실제로 위로? (아래로면 펌웨어 DIR(PA7) 반대)
- 행정 끝(0 / z_max=0.5m) 소프트리밋이 실제 기구 범위와 맞는지. 넘으면
  LIFT_MAX_STEPS / z_max 조정. 처음엔 행정 좁게 잡고 늘리기.

## 6. 통합 — FOLLOW 첫 주행
위 1~5 다 맞은 뒤: 주인 서서 FOLLOW(모드1) → 거리/각도 유지하는지.
- 떨리면 control 게인(kp_pos/kp_byaw) 낮추기(realism_check 참고).
- yaw 누적이 한 방향으로 쌓이면(공전/회전 많이) 2바퀴 한계 주의(언와인딩 미구현).

## 7. ★안전
- 첫 통전은 바퀴 들고. v_max 0.4(control)·V_MAX 0.4(펌웨어) 양쪽 제한 확인.
- 워치독: /control_cmd 끊기면 0.3s 후 휠 정지(bridge+펌웨어 양쪽). 비상시 노드 Ctrl+C.
