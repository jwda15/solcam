# 우분투22 / ROS2 Humble 전체 브링업 가이드

윈도우 프리뷰는 UI 확인용이었고, 여기서는 손동작 인식 → 제어값까지 **진짜로**
돌린다. (OAK·실차 없이도 폰 카메라 한 대로 손동작 인식까지 검증 가능)

## 0. 사전 설치 (최초 1회)

```bash
# ROS2 Humble 은 이미 설치되어 있다고 가정 (source /opt/ros/humble/setup.bash)
sudo apt update
sudo apt install -y python3-colcon-common-extensions python3-opencv \
                    python3-pygame android-tools-adb scrcpy ffmpeg \
                    v4l2loopback-dkms v4l-utils

# 손동작 인식(HaGRID YOLO) — ultralytics(+torch)
pip install ultralytics            # conda 환경이면 conda 끄고: conda deactivate

# 제스처 모델 다운로드 (레포에 없음, 라이선스 때문에 gitignore)
cd <repo>/ros2_gesture_node/models && bash download.sh    # YOLOv10n_gestures.pt
```

> conda가 켜져 있으면 colcon/ament가 깨진다(`No module named 'em'`).
> 빌드/실행 전 반드시 `conda deactivate`.

## 1. 빌드

경로에 공백·한글 있으면 colcon 메시지 생성이 깨지므로 영문 워크스페이스에 심볼릭 링크.

```bash
source /opt/ros/humble/setup.bash
# ByteTrack 라이브러리
cd <repo>/build && cmake .. && cmake --build . -j

mkdir -p ~/solcam_ws/src && cd ~/solcam_ws/src
for p in ros2_tracking_node ros2_control_node ros2_gesture_node ros2_phone_bridge; do
  ln -sfn <repo>/$p $p
done
cd ~/solcam_ws
colcon build --symlink-install        # tracking 먼저(또는 함께) — 메시지 의존
source install/setup.bash
```

## 2. 카메라 한 대로 손동작 인식까지 (OAK 없이 검증)

phone_bridge 가 발행하는 `/phone/image` 를 gesture_node 입력으로 재사용한다.
폰을 USB로 꽂거나, 노트북/USB 웹캠을 써도 된다.

### (A) 폰 카메라로
```bash
sudo <repo>/ros2_phone_bridge/scripts/setup_v4l2loopback.sh 2     # /dev/video2
<repo>/ros2_phone_bridge/scripts/start_scrcpy_camera.sh /dev/video2  # 폰 후면카메라
ros2 launch ros2_phone_bridge phone.launch.py video_device:=/dev/video2
```

### (B) 노트북/USB 웹캠으로 (폰 없이 손동작만 빨리 보고 싶을 때)
```bash
# phone_bridge 가 /dev/video0 을 그대로 읽어 /phone/image 로 발행 (adb 배터리는 경고만)
ros2 run ros2_phone_bridge phone_bridge --ros-args -p video_device:=/dev/video0
```

→ 둘 다 `/phone/image`, `/phone/battery`, `/phone/recording` 발행.

## 3. 손동작 인식 + UI

```bash
# gesture 입력을 /phone/image 로 지정 (OAK 쓰면 image_topic 생략 → /oak/rgb/image_raw)
ros2 launch ros2_gesture_node gesture.launch.py \
     recognizer:=hagrid ui:=true \
     --ros-args -p gesture_node.image_topic:=/phone/image
# 개발 PC면 UI 전체화면 끄기: gesture_ui_node.fullscreen:=false (yaml 또는 -p)
```

따봉(1.5초) → 메뉴 → 1~4 선택, 거꾸로따봉=뒤로. UI에 폰 영상+메뉴가 뜬다.

## 4. 제어 노드 (제스처가 만든 제어값 확인)

```bash
ros2 launch ros2_control_node control.launch.py        # IDLE 시작
```

확인용 토픽 echo (다른 터미널):
```bash
ros2 topic echo /control_mode      # Mode→Follow 선택 시 data: 1
ros2 topic echo /adjust_cmd        # Wheel/Lift 조절 시 param/value 스트림
ros2 topic echo /gesture_active    # 손동작 세션 동안 true (몸체 감속정지)
ros2 topic echo /phone/recording   # Other→Rec 시 true/false
ros2 topic echo /control_debug     # 선분 D·φ, 주인 글로벌추정, 발행 명령
```

손동작 → 토픽 값 매핑(빠른 점검):
- Mode → Follow/Rotate/Idle  ⇒ `/control_mode` 1 / 2 / 0
- Wheel → Farther/Closer/Pan ⇒ `/adjust_cmd` (SEG_DISTANCE / HEADING_OFFSET), 꾹 누르면 연속
- Lift → Up/Down             ⇒ `/adjust_cmd` (LIFT_HEIGHT)
- Other → Rec                ⇒ `/phone_cmd` record_toggle → 폰 녹화 → `/phone/recording`
- Other → Zoom/Focus         ⇒ `/phone_cmd` (현재 phone_bridge 에서 로그만)

## 6. 모델 없이 배선만 빠르게
```bash
ros2 launch ros2_gesture_node gesture.launch.py recognizer:=mock
ros2 topic pub --once /gesture_mock std_msgs/String "data: like"   # 따봉 주입
ros2 launch ros2_phone_bridge phone.launch.py mock:=true           # 합성영상+가짜배터리
```

## 자주 나는 문제
- `No module named 'em'` / colcon 깨짐 → `conda deactivate` 후 재빌드.
- ultralytics import 느림/CUDA 경고 → CPU로도 동작. Jetson은 JetPack torch 사용.
- `/dev/video2` 없음 → setup_v4l2loopback.sh 다시(부팅마다). `v4l2-ctl --list-devices` 확인.
- adb 배터리 경고 → 폰 USB 디버깅 ON + `adb devices` 승인. 웹캠 모드만이면 배터리는 안 옴.
- 제스처 인식 안 됨 → `ros2 topic hz /phone/image` 로 프레임 들어오는지, conf_threshold 낮춰보기.
