## 1. 한 줄 상황
solcam = 1인 크리에이터용 팔로우 카메라 로봇(OAK-D 추적 + 메카넘 + 리프트 +
상단yaw 카메라 + 안드로이드 폰 촬영 + 손동작 UI). 직전에 **ros2_driver_bridge
(/control_cmd→STM32 UART, /odom·/top_yaw_state 발행) + STM32 펌웨어(driver_firmware)
+ 손동작 Bearing(공전) 메뉴**가 GitHub에 합쳐졌다. 로컬도 그 최신(커밋 9947cfe)으로 동기됨.
이제 **Jetson에서 첫 colcon 빌드**. ★이번이 control_node 의 새 C++(모드3 FOLLOW2,
모드4 ORBIT, 센서 QoS, owner 끊김 게이팅)의 첫 실제 컴파일 → 빌드에러 나오면 잡을 것.

## 2. 드라이버 통합 점검 결과 (이미 검증됨 — 다시 안 해도 됨)
bridge ↔ 펌웨어 ↔ control_node 계약을 정밀 교차검증 완료:
 - 메카넘 역기구학(펌웨어) ↔ 정기구학(bridge) 정확히 역관계, 상수 동일(R0.05/LX0.36/LY0.26)
 - UART 28B(TX)/19B(RX) 오프셋·체크섬(sum&0xFF) 바이트단위 일치(펌웨어 reserved 0 패치됨)
 - 엔코더: 펌웨어가 읽고 즉시 0리셋 → "주기당 델타" ↔ bridge 적분 가정 일치
 - odom 필드(pose x/y/yaw쿼터, twist wz) ↔ control odomCallback 추출 일치
 - QoS: bridge /odom·/top_yaw_state = BEST_EFFORT ↔ control 구독도 BEST_EFFORT → 호환
 - V_MAX 0.4 = control v_max 0.4, 워치독 300ms 양쪽. bridge test_protocol 52/52 PASS.
★남은 건 코드가 아니라 **하드웨어 캘리브**(5번 참고): encoder_cpr(PLACEHOLDER),
  엔코더 순서/부호, DOF 방향, 상단yaw 0점/부호.

## 3. 지금 할 일 — 순서
### 3-1. 상태 진단 (제일 먼저, 사용자에게 실행 요청)
```bash
echo "ROS: $ROS_DISTRO"; which colcon; python3 -V
echo "conda: $CONDA_DEFAULT_ENV"                 # 비어야 함
ls ~/solcam ~/solcam_ws 2>/dev/null
git -C ~/solcam remote -v 2>/dev/null | head
python3 -c "import serial,cv2; print('serial/cv2 OK', cv2.__version__)" 2>&1 | tail -1
python3 -c "import depthai; print('depthai', depthai.__version__)" 2>&1 | tail -1
ls /opt/ros/humble/setup.bash 2>/dev/null && echo "Humble 있음"
```
### 3-2. 코드 받기
- repo 있으면: `cd ~/solcam && git pull myrepo main` (remote 이름 위에서 확인; origin이면 origin)
- 없으면: `git clone https://github.com/jwda15/solcam.git ~/solcam`
### 3-3. 빌드 (conda 끄고, 영문 워크스페이스)
```bash
conda deactivate 2>/dev/null
source /opt/ros/humble/setup.bash
REPO=~/solcam
# ByteTrack 라이브러리(libbytetrack.so)
cd "$REPO/build" && cmake .. && cmake --build . -j
# 워크스페이스 심볼릭 링크 (영문 경로!)
mkdir -p ~/solcam_ws/src && cd ~/solcam_ws/src
for p in ros2_tracking_node ros2_control_node ros2_gesture_node \
         ros2_phone_bridge ros2_driver_bridge; do ln -sfn "$REPO/$p" "$p"; done
# (depthai 설치돼 있을 때만) ln -sfn "$REPO/ros2_yolo_oak" ros2_yolo_oak
cd ~/solcam_ws
colcon build --symlink-install   # tracking 메시지가 먼저 빌드되어야 함(자동 정렬)
source install/setup.bash
```
### 3-4. 빌드 검증 포인트 (★새 코드 첫 컴파일)
 - C++ 새 파일: `follow2_controller.cpp`, `orbit_controller.cpp` 컴파일 통과?
 - `control_node.cpp`: `rclcpp::SensorDataQoS()`(/odom·/top_yaw_state·/proximity),
   `Mode::FOLLOW2`·`Mode::ORBIT` case, adjust 라우팅 — 에러 없는지.
 - 빌드 후: `ros2 pkg executables ros2_driver_bridge` 등으로 실행파일 등록 확인.
 - (오프라인 로직검증은 이미 통과: g++ T1~T15, menu 12, bridge 52. 여긴 colcon 확인용.)

## 4. ★Jetson 특화 주의
 - **aarch64**: `pip install torch/ultralytics` 가 x86용을 끌어와 깨질 수 있음 →
   gesture 인식 런타임은 JetPack용 torch 별도. ★단 **colcon 빌드엔 torch/depthai/
   pygame 불필요**(전부 런타임 의존) → 빌드는 그것 없이도 됨.
 - **driver_firmware/ 는 colcon 빌드 대상 아님** — STM32 펌웨어다(CubeIDE에서 빌드→
   보드 플래시). package.xml 없어 colcon이 자동 무시.
 - **conda off**, 경로 한글/공백 금지(→영문 워크스페이스 심볼릭).
 - depthai 3.x(oak_detector v3 API). OAK 런치는 repo 루트 cwd에서(.depthai 캐시).
 - 런타임 빠진 것만 apt: `python3-serial`(bridge), `python3-opencv`,
   `ros-humble-cv-bridge`(oak), `python3-pygame`(ui/teleop).

## 5. 빌드 다음 (실기 — 하드웨어 캘리브)
 - 드라이버 브리지: `ros2 run ros2_driver_bridge driver_bridge --ros-args
   -p port:=/dev/ttyUSB0`(핀UART면 /dev/ttyTHS1). pyserial + dialout 권한 필요.
 - ★encoder_cpr 실측 후 교체(휠1회전당 카운트, 기어×쿼드x4). 안 맞으면 odom 스케일 틀림.
 - 엔코더 순서/부호(enc1~4 → FL/FR/RL/RR), DOF 방향(+vx전진/+vy좌/+wz CCW): IDLE

 - 상단yaw 0점/부호(펌웨어 636.6 steps/rad)가 control theta_head 규약과 같은지.
 - 전체 기동: `scripts/solcam.sh run` (OAK→gesture→phone→ui; driver_bridge는 따로).
