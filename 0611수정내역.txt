[★주행 모드 5종 — 동작 + odom 필요 여부]  (odom = 팀원 드라이버가 줘야 함)
 0 IDLE   : 정지 + 키보드 teleop. 상단yaw 정지.          odom 불필요
 1 FOLLOW : 선분(거리D+글로벌각φ) 유지, 진입시 캡처.       ★odom 필요
 2 ROTATE : 위치 고정, 몸체 yaw만 주인 추종.              ★odom 필요
 3 FOLLOW2: leash — 거리만 유지(단방향). 멀면 끌려가고     odom 불필요
            가까우면 후퇴 없이 정지. 구도 무시(wz=0). 데드존 널널.
 4 ORBIT  : 공전 — 반지름 유지하며 천천히 CCW로 돌기.      odom 불필요
            반지름=진입(선택) 시점 거리. 게걸음(wz=0).
 → FOLLOW/ROTATE 는 /odom 없으면 몸체속도 0(상단yaw·리프트만 나감).
   IDLE/FOLLOW2/ORBIT 는 /odom 없이도 동작 → 드라이버 전 실기 테스트 가능.

[★상단 yaw(카메라)]
 - FOLLOW/ROTATE/FOLLOW2/ORBIT 모두 동일 trackTopYaw + 동일 파라미터로
   같은 속도 락온. IDLE 만 의도적으로 정지.
 - 언와인딩 미구현(theta_soft_max≈무한): ORBIT 여러 바퀴 돌면 한 방향 누적 →
   실기에서 케이블/슬립링 한계 보고 제한 걸 것.

[★제어값 — 현실성/튜닝]
 - v_max 0.4 m/s 유지(휠 토크 빠듯, 어차피 빠른 사람은 못 따라감).
 - 재튜닝 후: 0.5m 오차→0.35m/s, 회전 17°→0.30rad/s. 큰 오차도 0.4에서 클램프.
 - kd 는 일부러 안 올림(OAK depth 노이즈를 D항이 증폭→화면 떨림).
 - 튜닝은 "값이 틀려서"가 아니라 감(感): 굼뜨면 kp↑, 출렁이면 kd 살짝↑.
   숫자 확인 도구: ros2_control_node/test/realism_check.cpp (g++).

[★폰 녹화 (요청 구조)]

[★운영 규율 (두 번 데인 부분)]

   → "내가 띄우면 검은화면". 코드 버그 아님. 꼬이면 solcam.sh clean.
 - GUI 노드는 DISPLAY=:0 XAUTHORITY=... 필요(solcam.sh 가 처리).
