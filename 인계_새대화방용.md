# solcam 프로젝트 인계 메모 (새 Cowork 대화방에 붙여넣기용)

> 우분투22(ROS2 Humble)에서 새 Cowork 방을 열 때 이 내용을 첫 메시지로 붙여넣으면
> 맥락이 이어진다. 작업 폴더로 이 repo(solcam)를 mount 할 것.
> 윈도우로 돌아가면 기존 방에서 계속하면 된다.

## 1. 프로젝트 한 줄
1인 크리에이터용 **팔로우 카메라 로봇 "solcam"** (캡스톤). OAK-D로 주인을 추적하고
메카넘 차체+리프트+상단 회전 스테이지로 따라다니며, **폰을 촬영 카메라**로 쓴다.
손동작(HaGRID 제스처)으로 모드/구도/리프트/촬영을 제어. 7" LCD UI.

## 2. 하드웨어 / 환경
- 추론·제어: Jetson Orin Nano Super, Ubuntu 22.04 + ROS2 Humble (실기, 며칠 뒤)
- 카메라: OAK-D S2 (RVC2) — YOLOv8n .blob 온보드, SpatialDetection → bbox+XYZ
- 촬영 카메라: 안드로이드 폰(삼성 Quantum4), USB로 Jetson 연결
- 모터: 휠 JGB37-520(메카넘4), 리프트 NEMA23+DM542, 상단yaw NEMA17+A4988
- 개발: 윈도우(프리뷰만) + 우분투22(실제 빌드/실행). git remote 는 `myrepo`
  (origin은 ByteTrack 원본이므로 push 금지). 커밋 author = jwda15 / whwjddn6318@gmail.com

## 3. 패키지 구조 (repo 루트)
- `src/`,`include/ByteTrack/` : ByteTrack-cpp 본체(빌드→libbytetrack.so).
  원본 대비 Kalman 8D→10D(z=depth), 1차매칭 IoU→3D Mahalanobis.
- `ros2_tracking_node/` : /detections+/camera_info → /owner_pose(주인 3D)
- `ros2_control_node/`  : C++(ament_cmake). /owner_pose+/odom → 6자유도 명령.
  - 헤더=선언/ src=정의. 튜닝값 전부 `include/control_node/params.hpp`
  - 전략패턴: IController + controllerFor(mode). ControllerBase 공통.
  - 모드0 IDLE(키보드 teleop), 모드1 FOLLOW(선분유지), 모드2 ROTATE(제자리회전)
  - 제어: PD + 데드존 + 가속(슬루)제한 + 발행직전 v_max 0.4 클램프
  - 손동작: /control_mode, /adjust_cmd(거리/헤딩/리프트), /gesture_active(몸체 감속정지)
  - ROS 없는 시뮬테스트 test/test_sim.cpp (T1~T13 통과)
- `ros2_gesture_node/`  : ament_python. 손동작 supervisor + LCD UI
  - menu.py(상태기계, ROS무관, 단위테스트 12건), recognizer.py(HaGRID YOLO/mock),
    gesture_node.py(인식→토픽), hud.py(렌더), ui_node.py(LCD), tools/ui_preview*.py(PC프리뷰)
  - 메뉴 트리: 따봉(1.5s)→메뉴 / 1~4 선택 / **거꾸로따봉(dislike)=뒤로**
    1단: **Mode / Wheel / Lift / Other**
      Mode: Follow/Rotate/Idle (→/control_mode 1/2/0)
      Wheel: Farther/Closer/Pan L/Pan R (→/adjust_cmd, 꾹=연속jog)
      Lift: Up/Down (→/adjust_cmd, 연속)
      Other: Phone(Zoom+/Zoom-/Focus) / OAK view / Power off / **Rec(녹화토글)**
  - UI 디자인: 영상 풀스크린 배경 + 하단 카드 독. 테마 흰70/파랑20/검정10,
    포인트 블루 #1e90ff. 단발=테두리 좌→우 파랑 차오름 후 0.1s 흰 반짝.
    연속(jog)=적용 순간부터 카드 계속 흰색. 따봉/거꾸로따봉도 작은 파란 게이지.
    상단 좌=모드, 우=녹화시간(REC mm:ss)+배터리. Rec 카드는 상태따라 ON/OFF.
- `ros2_phone_bridge/`  : ament_python. 안드 폰 USB 브리지(0606 신설)
  - scrcpy(후면카메라)→v4l2loopback(/dev/videoN)→cv2→/phone/image
  - adb dumpsys battery → /phone/battery (BatteryState)
  - /phone_cmd record_toggle → ffmpeg 로 v4l2 녹화→종료시 adb push로 폰 DCIM 전송
    → /phone/recording. zoom/focus는 자리만(로그). mock 모드 있음.
  - 셋업 스크립트 scripts/setup_v4l2loopback.sh, start_scrcpy_camera.sh
- `ros2_yolo_oak/`(실기 OAK 검출), `ros2_yolo_d435i/`(검증용), `tum_publisher.py`

## 6. 우분투에서 할 일 (다음 단계)
**루트 `우분투_브링업.md` 그대로 따라가면 됨.** 핵심 순서:
1. deps 설치: `pip install ultralytics`, apt: python3-opencv python3-pygame
   android-tools-adb scrcpy ffmpeg v4l2loopback-dkms v4l-utils. (conda는 deactivate)
2. 모델: `ros2_gesture_node/models/ && bash download.sh` (YOLOv10n_gestures.pt, gitignore)
3. colcon build (영문 워크스페이스 심볼릭링크), tracking 먼저
4. 카메라: phone_bridge가 /phone/image 발행 → gesture는 image_topic:=/phone/image 로 재사용
   (OAK 없이 손동작 인식 검증 가능). 노트북 웹캠이면 video_device:=/dev/video0
5. gesture_node+ui_node 실행 → control_node 실행 →
   `/control_mode`,`/adjust_cmd`,`/gesture_active`,`/phone/recording`,`/control_debug` echo로 확인
**검증 목표: 손동작 → 메뉴 → 선택 → 해당 제어값 토픽이 실제로 나오는지.**
그 다음: 줌/포커스 실제 구현, 실차 모터 드라이버 연동, Jetson 배포.

## 9. ★0607 업데이트 — 우분투 실기 검증됨 + 운영법 (이 섹션이 최신)

### 이미 검증 완료 (다시 안 해도 됨)
- colcon build (Humble, 4~5개 패키지) 성공. apt 런타임 의존성 설치됨
  (pygame/adb/scrcpy/ffmpeg/v4l-utils, cv_bridge). v4l2loopback 은 distro 모듈로 동작
  (dkms 불필요, 커널 6.8에서 dkms 빌드 실패는 무시).
- OAK 검출+ByteTrack 3D 추적 실기 확인: is_detected/track_id 안정, 깊이 스케일 정상
  (0.37m→3.2m 따라옴). 2~3m 실제 따봉 인식됨. mock 체인(/control_mode=1)도 PASS.
- 이분할 UI(PHONE|OAK), QoS 수정(OAK BEST_EFFORT 구독), X버튼 종료 수정 — 동작 확인.

### ★반드시 알아야 할 교훈 (0607 세션이 헤맸던 진짜 원인)
- **코드 버그 아니었음.** ui_node/hud/phone_bridge 코드는 정상(빌드본=소스 확인).
- "폰 영상 멈춤"·"내가 띄우면 검은화면"의 진짜 원인은 **kill -9 남발**:
  - scrcpy 를 -9 로 죽이면 → /dev/videoN 포맷 잠김(VIDIOC_G_FMT) → 폰 영상 죽음.
  - ROS 노드를 -9 로 죽이면 → /dev/shm/fastrtps_* 누적 → DDS 합류 실패(검은화면).
  - "봇이 띄우면 되고 내가 띄우면 안 됨"은 코드 차이가 아니라 그 순간 DDS 그래프
    합류 상태 차이(봇이 -9로 망가뜨려 놓은 것). **절대 kill -9 쓰지 말 것.**
- split 규칙: `split = (메뉴 MENU 상태) OR oak_view`. 메뉴 중엔 항상 이분할이라
  oak_view 토글로 안 풀림 — 버그 아니고 설계.

### ★시작/종료는 이걸로만 (scripts/solcam.sh)
```bash
# 본인 환경에 맞게(예시는 0607 PC): repo·scrcpy 경로
export SOLCAM_REPO=<repo> SOLCAM_WS=~/solcam_ws
export SCRCPY_DIR=$HOME/scrcpy_bin/scrcpy-linux-x86_64-v4.0   # scrcpy>=2.0 prebuilt 필요

scripts/solcam.sh clean    # 꼬였을 때 1회: 노드 graceful 종료 + shm 정리 + v4l2 리로드(sudo)
scripts/solcam.sh run      # OAK→gesture(/phone/image)→phone_bridge(managed)→ui 순서 기동
scripts/solcam.sh stop     # graceful 종료(절대 -9 금지)
scripts/solcam.sh status   # 노드/장치/adb/shm 점검
```
- GUI 노드(ui_node)는 `DISPLAY=:0 XAUTHORITY=/run/user/$(id -u)/gdm/Xauthority` 필요
  (solcam.sh 가 자동 설정). 검은화면이어도 ~10초 디스커버리 대기, 바로 Ctrl+C 금지.
- OAK 런치는 repo 루트 cwd 에서(.depthai 캐시). scrcpy 는 2.0+ prebuilt 바이너리 사용.
- 폰: USB 디버깅 ON + `adb devices` 승인만 하면 됨(앱 불필요, 기종 무관).

### ★v4l2 꼬임 자동복구 (phone_bridge 에 내장됨, 0607 추가)
- phone_bridge 가 "scrcpy 살아있는데 10초+ 프레임 없고 포맷 미준비" 감지 시
  v4l2loopback 을 스스로 리로드해 복구(_tick_health). 파라미터 `v4l2_reset_cmd`,
  `wedge_timeout`. solcam.sh 가 자동으로 넘김.
- 완전 무인 복구하려면 1회 설정(선택): scripts/solcam.sudoers 안내대로
  reset 스크립트를 /usr/local/sbin 에 root 소유로 두고 NOPASSWD 허용.
  안 하면 solcam.sh 가 sudo 비번 한 번 물으며 동작.

### 남은 일 (우선순위)
1. solcam.sh run 으로 풀스택 띄워 화면 보며 따봉→이분할→Mode/Follow 선택까지 확인.
2. 줌/포커스 실제 구현(현재 phone_bridge 에서 로그만). 캡처=scrcpy 카메라.
3. control_node 실차 모터 드라이버(팀원) 연동, Jetson 배포.
