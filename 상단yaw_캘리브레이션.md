# 상단 yaw 캘리브레이션 / 펌웨어 메모

펌웨어(`D:/capstone/yaw`, STM32F407) 상단 yaw 스테이지 측정·검산 결과 정리.
재플래시·튜닝 때 이 파일만 보면 됨. (최종 업데이트: 2026-06-21)

## 핵심 수치

| 항목 | 값 | 근거 |
|---|---|---|
| 스텝 분해능 | **0.1125°/step** | 1.8° 스테퍼 × 1/16 마이크로스텝 = 3200 step/rev |
| 스텝 속도 | **250 steps/s** | TIM7 500Hz 인터럽트 ÷ 2(토글) |
| **회전 속도** | **28.125°/s (≈28°/s)** | 250 × 0.1125 |
| 회전 한계 | **±90° = ±800 steps** | `YAW_STEP_LIMIT 800` (선 꼬임 방지) |
| 90° 도달 시간 | **3.2초** | 800 ÷ 250 |

TIM7 클럭 체인: 16MHz(HSI=SYSCLK=APB1) ÷ 16(prescaler) ÷ 2000(period=1999) = 500Hz.

## 입력시간 → 회전각 (고정속도, 부호=방향, 90°에서 정지)

| 누른 시간 | 회전각 |
|---|---|
| 1초 | ~28° |
| 2초 | ~56° |
| 3초 | ~84° |
| 3.2초+ | 90° (한계, 자동 정지) |

θ(t) ≈ 28.125 × t [deg], 최대 90°.

## 제어 방식 (velocity 모드)

- `Yaw_SetTarget(target_rad, active)`: **부호 = 회전방향**, `|target| > YAW_DEADBAND(0.05 rad)`면 고정속도 회전, 이내면 정지. **절대각 추적 아님(개루프).**
- 우리 control_node가 보내는 값: **0.0(정지) 또는 ±1.0(방향)** 뿐 → 펌웨어 deadband와 무관.
- 중앙 정지 판정은 우리쪽 `az_dead`(현재 0.10 rad ≈ ±5.7°)가 결정. 28°/s에서 제어 1주기(20ms) 이동량 0.56° → 불감대 안에서 채터링 없이 정지.
- 방향이 실제와 반대면 `top_yaw_sign = -1.0` (params.hpp / control_params.yaml).
- 펌웨어 DIR 핀(PC3)은 반전 로직: `yaw_dir>0 → RESET`.

## UART 상태(TX) 포맷 — ★중요

현 펌웨어 TX(ASCII): `"e1,e2,e3,e4,y=<yaw_current_steps>\n"`
- 과거 `yr=<running flag>`에서 **`y=<실제 step 카운트>`로 바뀜.**
- driver_bridge ASCII 파서는 **앞 4개 정수(엔코더)만** 사용 → odom 정상, 5번째 필드 무시. **수정 불필요.**
- `y=` 값은 실제 yaw 위치(steps). steps × 0.1125° = 현재각. 필요시 driver_bridge에서 파싱해 `/top_yaw_state`(rad) 발행 가능(현재 미구현, 개루프라 불필요).

## 핀맵 (yaw)

- PC0 = STEP, PC3 = DIR, PC2 = ENA(active LOW)
- 리프트: PA6=STEP, PA7=DIR, PC4=ENA / `LIFT_STEPS_PER_MM 38.55`

## 주의 / TODO

- **±90° 한계는 "전원 켠 순간 위치" 기준** (호밍 없음). 안 중앙으로 켜지면 좌우 가용범위 비대칭 → 부팅 센터링/호밍 추후 고려.
- **28°/s가 빠른 측면 보행엔 다소 느릴 수** 있음 → 프레임 이탈 시 몸체 추종이 보완. 더 빠르게: TIM7 period↓(스텝레이트↑), 단 탈조(missed step) 확인.
- `yaw_current_steps`는 전원 재투입에만 리셋(개루프 누적). 드리프트 보정 없음 — 현 단계 OK.
- 펌웨어 `V_MAX=0.4`는 안전 천장. 실제 주행 상한은 우리 `v_max=0.15`가 더 낮게 클램프.
